# 종목 리서치 리포트 생성 — $ARGUMENTS

`$ARGUMENTS` 종목에 대해 역할별 애널리스트 서브에이전트를 협업시켜 리서치를 만들고,
검수를 통과하면 PPTX/PDF 리포트로 내보냅니다. 분석 관점은 **가치·펀더멘털 중심, 장기(3년+)** 입니다.

종목명이 모호하면(동명이종목 등) **위임 전에 종목코드/대상 시장을 한 번만** 되묻습니다.

## 실행 흐름

1. **3개 분석 병렬 실행** — Agent 도구로 아래 셋을 동시에 호출합니다.
   - `fundamental-analyst` — 재무제표·실적·공시 (DART OpenAPI, `DART_KEY` 필요)
   - `market-tech-analyst` — 주가·추세·이동평균·52주 고저·변동성·거래량 (FinanceDataReader, 키 불필요)
   - `news-sentiment-analyst` — 뉴스·이슈·공시 해석·시장 심리 (웹서치)

2. **종합·리스크 점검** — 위 세 결과를 `risk-manager-synthesizer`로 모아 핵심 리스크 3가지와
   시총·거래대금 등 규모/유동성(pykrx)을 더합니다.

3. **리포트 작성** — 종합 결과를 아래 **5부 구성**으로 `reports/$ARGUMENTS.md`에 저장합니다.
   1. 표지 — 종목명·기준일·작성 관점
   2. 재무 — 재무제표·밸류에이션·성장성/수익성 지표
   3. 차트 — 가격 추세·거래량·주요 지표 시각화용 데이터
   4. 리스크 — 시장/산업/기업 고유 리스크와 발생 가능성·영향도
   5. 종합의견 — 핵심 논거와 판단 근거 정리

4. **검수** — `verification-analyst`로 `reports/$ARGUMENTS.md`를 점검합니다(정확성·일관성·완결성·근거/형식).
   에이전트는 **리포트를 직접 고치지 않고** 문제 표와 **통과/보류** 판정만 냅니다.
   - **보류**가 나오면 지적된 항목을 수정한 뒤 **재검수**합니다.
   - **통과** 시 다음 단계로 진행합니다.

5. **내보내기** — `report-pptx` 스킬로 `reports/$ARGUMENTS.md`를 PPTX/PDF로 변환합니다.
   ```
   python "${CLAUDE_PLUGIN_ROOT}/skills/report-pptx/build_pptx.py" reports/$ARGUMENTS.md --pdf
   ```
   완료되면 생성된 파일 경로(`reports/$ARGUMENTS.pptx`, `reports/$ARGUMENTS.pdf`)를 사용자에게 알립니다.
   (PPTX를 열 오피스가 없는 환경에서는 PDF가 브라우저에서 바로 열립니다.)

## 작성 규칙
- 문체는 **"~입니다" 체**를 사용합니다.
- **모든 수치에는 출처와 기준일을 함께 적습니다.** (예: "영업이익률 12.3% (출처: 2024 사업보고서, 기준일 2024-12-31)")

## 가드레일 (모든 산출물에 항상 적용)
1. **출처 없는 수치는 쓰지 않습니다.** 모든 수치 옆에 `(출처: 데이터명, 연도/날짜)`를 표기합니다.
2. **수치를 지어내지 않습니다.** 데이터를 못 구하면 **"확인 불가"**, 출처 없는 뉴스·루머는 **"미확인"** 으로 표기합니다.
3. **투자 행동을 단정하지 않습니다.** 매수·매도·보유, 목표가, 비중 확대/축소 표현을 금지하고, 판단 근거 정리(의사결정 지원)까지만 합니다. 최종 판단은 사람의 몫입니다.
4. **학습용 고지 + 출처 목록.** 리포트 첫머리에 "무료 공개 데이터 기반 학습용" 한 줄, 끝에 사용한 데이터 출처·기준일 목록을 넣고, 투자 자문이 아님을 명시합니다.

> 비밀값 주의: `DART_KEY` 등 API 키는 사용자의 `.env`에서 읽으며, 키를 하드코딩하거나 출력하지 않습니다.
