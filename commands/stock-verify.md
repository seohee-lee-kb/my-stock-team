# 리포트 검수 — $ARGUMENTS

`$ARGUMENTS`에 해당하는 완성된 리포트(`reports/$ARGUMENTS.md`, 또는 직접 지정한 경로)를
`verification-analyst` 에이전트로 품질 점검합니다.

1. 대상 리포트 경로를 확인합니다. 모호하면 종목/경로를 한 번 되묻습니다.
2. Agent 도구로 `verification-analyst`를 호출해 아래 4축으로 점검합니다.
   - **정확성** — 수치가 데이터와 맞고 계산·단위 오류는 없는가
   - **일관성** — 본문·표·결론이 서로 어긋나지 않는가
   - **완결성** — 빠진 항목 없이 네 분석이 다 담겼는가
   - **근거·형식** — 수치마다 출처(연도)가 있고, 매수/매도 단정 없이 양식을 지켰는가
3. 산출물(문제 표 + **통과/보류** 판정)을 그대로 사용자에게 전달합니다.

검증 애널리스트는 **리포트를 직접 수정하지 않습니다.** "보류"가 나오면 지적 항목을 수정한 뒤
`/my-stock-team:stock-verify`로 재검수하거나, 전체 흐름은 `/my-stock-team:stock-report`로 진행합니다.
